# 编码实现反错误模式

## 代码实现基本原则

- 代码应兑现建模报告中的算法、约束和参数。若必须替代，要在结果记录里明确说明原因、替代方案和影响。
- 先保证可行解，再追求优化目标。任何输出方案都要逐条回代检查约束。
- 随机算法要固定随机种子，并用多次运行或稳定性指标说明结果可信度。
- 结果文件应记录关键参数、核心数值、约束检查、灵敏度或鲁棒性结果，以及可复现运行方式。
- 所有论文会引用的数值必须能追溯到结果记录、结果 JSON、表格或代码输出；不要在论文阶段重新估算或换一套四舍五入口径。
- 数据型图表应由真实结果数据生成，并保存图表对应的数据来源；概念流程图和路线图不应冒充数据图。

## 14 种编码常见错误

### 1. scipy.optimize.minimize 方向错误

`scipy.optimize.minimize` 只做最小化；最大化利润、覆盖率或得分时要对目标函数取负，并在结果记录中还原真实目标值。

❌ 错误：
```python
result = minimize(lambda x: profit(x), x0)  # profit 是正数，minimize 会找最小利润
```

✅ 正确：
```python
result = minimize(lambda x: -profit(x), x0)  # 取负，找最大利润
real_profit = -result.fun  # 还原真实值
```

### 2. scipy 不等式约束方向错误

`scipy` 不等式约束方向是 `fun(x) >= 0`；容量约束、下界约束、预算约束最容易写反，写完要代入几个边界点检查符号。

❌ 错误：
```python
# 想表达 x <= 100，但写成了 100 - x <= 0（方向反了）
{'type': 'ineq', 'fun': lambda x: x - 100}
```

✅ 正确：
```python
# x <= 100 转化为 100 - x >= 0
{'type': 'ineq', 'fun': lambda x: 100 - x}
```

### 3. 盲目相信求解器 success

不要只相信求解器的 `success`。最优解必须重新代入所有约束函数，输出每条约束的值、边界、松弛量和是否活跃。

```python
for i, constraint in enumerate(constraints):
    val = constraint['fun'](x_opt)
    print(f"约束 {i}: 值={val:.6f}, {'满足' if val >= 0 else '违反'}")
```

### 4. 整数变量未取整修复

整数变量、0-1 变量、人数、车辆数、批次数等不能停留在连续解。取整后必须重新验证可行性；不可行时用明确的修复启发式，而不是直接四舍五入。

```python
x_int = np.round(x_cont).astype(int)  # 取整
# 验证可行性
if not is_feasible(x_int):
    x_int = repair_heuristic(x_int)  # 修复启发式
```

### 5. 物理参数不一致

物理参数、单位和符号必须与建模报告一致。代码里不要出现与报告不一致的长度、质量、速度、半径、时间步长或坐标原点。

### 6. 数据读取未检查

数据读取后先检查编码、列名、形状、单位、缺失值和异常值。不要在数据错位或列名乱码时继续建模。

```python
df = pd.read_csv("data.csv")
print(f"形状: {df.shape}")
print(f"列名: {df.columns.tolist()}")
print(f"缺失值:\n{df.isnull().sum()}")
print(f"数据类型:\n{df.dtypes}")
```

### 7. 数据泄露

预测代码要防数据泄露：划分数据后再 fit scaler/encoder；时序问题不能随机打乱；测试集不能参与调参。

❌ 错误：
```python
scaler.fit_transform(X_all)  # 全集 fit，泄露测试集信息
```

✅ 正确：
```python
X_train, X_test = train_test_split(X, test_size=0.2)
scaler.fit(X_train)  # 只在训练集 fit
X_train_scaled = scaler.transform(X_train)
X_test_scaled = scaler.transform(X_test)
```

### 8. 图论方向性错误

图论代码要检查节点数、边数、有向性、负权边、连通性和容量；最大流实现要有残量网络或等价机制。

### 9. 几何中心距替代实体碰撞

几何代码要避免中心距替代实体碰撞；遮蔽/覆盖类函数的参数应包含完整几何尺寸，离散采样近似要做采样数收敛检查。

### 10. 数值代码未检查收敛

数值代码要检查矩阵条件数、收敛状态、迭代次数、步长减半结果、随机种子复现性和边界输入。

### 11. 结果记录不完整

JSON 或结果表必须保存每个子问题的关键数值、约束检查、图表数据来源和可复现参数；不要只把最终一句结论写进 Markdown。

### 12. 多重共线性未检测

回归问题中 VIF > 10 说明严重共线，应用岭回归或删除高相关特征。

### 13. 标准化 scaler 全集 fit

时序数据或任何预测任务中，scaler 必须只在训练集 fit。全集 fit 会泄露未来信息。

### 14. 启发式算法单次运行

启发式算法（GA/SA/PSO/NSGA-II）必须固定随机种子、多次独立运行（≥5 次）、报告均值和标准差。单次运行结果不可信。

## 数据泄露防范规则

### 时序特征

- 用 `shift(1)` 获取上一期，禁止 `shift(-1)`
- 滚动特征：`rolling(w).mean().shift(1)` 排除当期
- 标准化：只用训练集 fit，测试集 transform
- 目标编码：只用训练集计算统计值

### 特征工程

- 滞后特征用 `shift(1)` 避免泄露
- 滚动窗口特征带 `shift(1)` 排除当期
- 分类变量用 One-Hot 或 Label Encoding
- 右偏分布考虑对数变换 `np.log1p()`

### 参数记录要求

所有关键参数必须有来源说明（数据统计/文献引用/网格搜索三选一），在代码注释或 print 中说明参数选择依据。

## 优化类工程约束

### 设计变量必须设定物理上下界

优化不能只求数学极值，必须检查实际物理可行性。常见致命错误：桌面缩尺模型（高度仅几百mm）的优化结果给出数米长的构件。

- **每个优化变量必须有上界和下界**，写清约束来源（几何限制/物理限制/题目要求）
- 如果无约束解违反物理限制，**大方在 print 中写出对比**：「无约束解为 XX，但其物理不可行（如构件超出模型高度），因此引入约束 XX ≤ XX_max，约束下最优解为 YY」
- 评委看到这种工程思维分析会给高分

### Q4 型结构优化问题特别注意

- 绳长 L 有几何上限（受模型离地高度限制），如 L ≤ 500mm 或 L ≤ 中心塔总有效高度
- 转速 n 有下限（不能为 0，设备需正常运行），如 n ≥ 0.3 r/s
- 构件长度有几何协调性约束

## 数据预处理规范

### 先判断题目类型

- **物理/力学机理题**（参数为题目给定的确定常量，如 H=200mm, m=3kg）：不要画直方图、箱线图或提「异常值清洗」「缺失值」——评委会认为你在套数据分析模板。EDA 聚焦于：打印关键参数表格 → 几何关系计算 → 量纲验证 → 物理一致性检查。
- **数据驱动题**（真的有数据集，有多个样本/分布）：执行以下 EDA 流程。

### 数据驱动题的 EDA 必须覆盖

1. `.info()` 和 `.head()` 查看数据结构
2. 缺失值报告：列出缺失数、缺失率、填充策略及理由
3. 异常值检测：IQR 或 Z-score，报告异常占比
4. 数据分布可视化：直方图/箱线图
5. 变量相关性分析：热力图
6. 分组对比分析

## 结果输出规范

### 每张图的绑图代码后必须 print 关键数据特征

这是因为 Agent 无法"看到"生成的图片，只能看到代码的文本输出。没有数据特征输出，后续写作手只能猜测图片内容，导致论文描述与图片不符。

### 时间序列图输出模板

```python
print("【图X数据特征 - 时间序列】")
print(f"   时间范围: {df['date'].min()} 至 {df['date'].max()}")
print(f"   起点值: {y.iloc[0]:,.2f}, 终点值: {y.iloc[-1]:,.2f}")
print(f"   整体趋势: {'上升' if y.iloc[-1] > y.iloc[0] else '下降'}")
print(f"   峰值: {y.max():,.2f}, 谷值: {y.min():,.2f}")
```

### 模型评估图输出模板

```python
print("【图X数据特征 - 模型拟合】")
print(f"   R²: {r2:.4f}")
print(f"   MAE: {mae:.4f}, RMSE: {rmse:.4f}, MAPE: {mape:.2f}%")
print(f"   拟合质量: {'优秀' if r2 > 0.9 else '良好' if r2 > 0.7 else '一般'}")
```

### 相关性热力图输出模板

```python
print("【图X数据特征 - 相关性】")
print(f"   最强正相关: {var1} vs {var2} (r={max_corr:.3f})")
print(f"   最强负相关: {var3} vs {var4} (r={min_corr:.3f})")
```

### 特征重要性图输出模板

```python
print("【图X数据特征 - 特征重要性】")
for i, (feat, imp) in enumerate(importance_df.head(5).values):
    print(f"   {i+1}. {feat}: {imp:.4f}")
```

### 预测图（含置信区间）输出模板

```python
print("【图X数据特征 - 预测结果】")
print(f"   点预测值: {prediction:,.2f}")
print(f"   95%置信区间: [{ci_lower:,.2f}, {ci_upper:,.2f}]")
```

### 混淆矩阵输出模板

```python
print("【图X数据特征 - 混淆矩阵】")
print(f"   总样本数: {cm.sum()}")
print(f"   总体准确率: {accuracy:.1%}")
```

### 结果汇总（每个子任务完成后必须输出）

```python

print("【本问题建模结果汇总】")
print(f"   模型类型: {model_name}")

print("=" * 60)
```
