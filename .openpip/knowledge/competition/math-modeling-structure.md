# 数学建模竞赛论文标准结构

## 概述

数学建模竞赛论文（如 MCM/ICM、CUMCM、美赛）有固定的结构规范。评委在短时间内评审大量论文，清晰的结构是拿高分的前提。

## 标准 7 章结构

### 第 1 章 问题重述与分析

**目的**：让评委快速理解问题背景和核心任务。

**内容要求**：
- 用 1-2 段话重述问题（不是复制题目，而是用自己的语言提炼）
- 明确问题的核心任务和子问题
- 给出问题的数学化描述
- 说明问题的实际意义和应用背景

**写作要点**：
- 不要直接复制题目原文
- 要体现对问题的深入理解
- 可以给出问题的层次化分解
- 用简洁的语言概括关键约束条件

**示例结构**：
```markdown
## 1.1 问题背景
[用 1-2 句话说明实际背景]

## 1.2 问题重述
[用自己的语言重述问题核心任务]

## 1.3 问题分析
[对问题进行层次化分解，明确各子问题的关系]
```

### 第 2 章 问题假设

**目的**：为建模提供合理的基础，展示思维的严谨性。

**内容要求**：
- 列出所有关键假设（通常 5-10 条）
- 每条假设附合理性论证
- 说明假设对模型的影响
- 标注哪些假设可以放松（用于灵敏度分析）

**假设分类**：
1. **必要假设**：建模必须的前提（如"忽略空气阻力"）
2. **简化假设**：降低问题复杂度（如"物体为质点"）
3. **理想化假设**：现实不可能但数学上可行（如"线性关系"）

**示例结构**：
```markdown
## 2.1 基本假设
1. 假设1：[内容]。合理性：[论证]
2. 假设2：[内容]。合理性：[论证]
...

## 2.2 假设影响分析
[说明哪些假设对模型结果影响较大，可用于灵敏度分析]
```

### 第 3 章 符号说明

**目的**：统一符号系统，便于阅读和复现。

**内容要求**：
- 用表格列出所有符号
- 包含：符号、含义、单位、取值范围（如适用）
- 符号命名遵循数学惯例（$x$ 表示自变量，$f$ 表示函数）
- 矩阵/向量用粗体

**示例结构**：
```markdown
## 符号说明

| 符号 | 含义 | 单位 | 取值范围 |
|------|------|------|----------|
| $x$ | 自变量 | m | $[0, 100]$ |
| $f(x)$ | 目标函数 | — | — |
| $\mathbf{A}$ | 系数矩阵 | — | $n \times n$ |
```

### 第 4 章 模型建立与求解

**目的**：核心章节，展示建模思路和求解过程。

**内容要求**：
- **模型建立**：从假设出发，逐步推导数学模型
- **模型求解**：给出求解算法、数值方法、工具
- **结果展示**：用图表清晰展示求解结果
- **结果分析**：对结果进行物理解释

**写作要点**：
- 逻辑清晰：假设→定义变量→建立关系→求解
- 公式推导完整：关键步骤不跳步
- 求解过程可复现：给出算法伪代码或工具说明
- 结果可视化：用图表而非纯文字展示

**常见模型类型**：
1. 优化模型（线性规划、非线性规划、整数规划）
2. 统计模型（回归分析、时间序列、主成分分析）
3. 微分方程模型（常微分、偏微分、差分方程）
4. 仿真模型（蒙特卡洛、Agent-based）
5. 机器学习模型（分类、聚类、预测）

**示例结构**：
```markdown
## 4.1 问题一的模型
### 4.1.1 模型假设
[针对子问题的具体假设]

### 4.1.2 模型建立
[推导过程，关键公式编号]

### 4.1.3 模型求解
[算法描述、工具、求解过程]

### 4.1.4 结果分析
[结果物理解释、图表展示]

## 4.2 问题二的模型
...
```

### 第 5 章 模型检验与灵敏度分析

**目的**：验证模型可靠性，展示鲁棒性。

**内容要求**：
- **模型检验**：用已知数据或特殊情况验证模型
- **灵敏度分析**：检验关键参数变化对结果的影响
- **误差分析**：讨论模型的精度和局限性

**灵敏度分析方法**：
1. 单参数灵敏度分析（改变一个参数）
2. 多参数灵敏度分析（同时改变多个参数）
3. 蒙特卡洛模拟（随机抽样）
4. 龙卷风图（参数影响排序）

**示例结构**：
```markdown
## 5.1 模型检验
### 5.1.1 检验方法
[描述验证方法]

### 5.1.2 检验结果
[验证结果和结论]

## 5.2 灵敏度分析
### 5.2.1 单参数分析
[改变单个参数的结果]

### 5.2.2 多参数分析
[同时改变多个参数的结果]

### 5.2.3 灵敏度结论
[参数敏感性排序、模型鲁棒性结论]
```

### 第 6 章 模型评价与改进

**目的**：客观评价模型优缺点，展示批判性思维。

**内容要求**：
- **优点分析**：模型的创新点、实用性、效率
- **缺点分析**：模型的局限性、简化带来的误差
- **改进方向**：未来可以如何改进
- **推广性讨论**：模型是否可以推广到其他场景

**示例结构**：
```markdown
## 6.1 模型优点
1. [优点1：如创新性、效率、精度]
2. [优点2]
...

## 6.2 模型缺点
1. [缺点1：如假设过于简化、适用范围有限]
2. [缺点2]
...

## 6.3 改进方向
1. [改进方向1]
2. [改进方向2]
...

## 6.4 推广性讨论
[模型是否可以推广到其他场景，需要哪些调整]
```

### 第 7 章 结论

**目的**：总结全文，给出最终答案。

**内容要求**：
- 简洁总结各问题的答案
- 概括主要创新点
- 说明模型的实际应用价值
- 不引入新内容

**示例结构**：
```markdown
## 7.1 问题一的答案
[简洁答案]

## 7.2 问题二的答案
[简洁答案]

## 7.3 主要创新点
1. [创新点1]
2. [创新点2]

## 7.4 应用价值
[模型的实际意义]
```

## 附录结构

```markdown
## 附录 A：代码清单
[关键算法的完整代码]

## 附录 B：数据表
[题目数据、处理后的数据]

## 附录 C：补充图表
[正文中未展示的图表]
```

## 参考文献格式

使用 GB/T 7714-2015 格式，按引用顺序编号。

## 常见结构错误

1. **❌ 问题重述直接复制题目**：应该用自己的语言提炼
2. **❌ 假设无合理性论证**：每条假设必须说明为什么合理
3. **❌ 符号说明缺失**：所有符号必须在首次使用前定义
4. **❌ 模型推导跳步**：关键公式推导不能省略
5. **❌ 缺少灵敏度分析**：竞赛论文必须包含灵敏度分析
6. **❌ 模型评价只说优点**：必须客观讨论缺点
7. **❌ 结论引入新内容**：结论只总结不展开
