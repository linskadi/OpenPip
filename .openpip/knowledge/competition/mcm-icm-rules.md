# MCM/ICM 竞赛专项规范

## 赛事规则

### 2026 年重要规则

- 论文总页数限制为 **25 页**（含摘要、正文、参考文献、附录，不含 AI 使用声明附录）。
- 允许使用 LLM 和生成式 AI 工具，但必须在附录中明确标注使用范围和校验方式；该附录不计入 25 页。
- Summary Sheet 是独立必要页面，评委权重极高；内容应涵盖方法、亮点和关键结论。

### 提交要求

- 论文必须使用 PDF 格式提交
- 团队控制号（Team Control Number）必须在 Summary Sheet 和每页页眉中出现
- 禁止在正文中出现任何学校名称、指导教师姓名或团队成员姓名（除控制号外）

## 评分关键维度

| 维度 | 权重 | 评分要点 |
| --- | --- | --- |
| 假设的合理性 | 高 | 假设必须服务于建模，不能是凑数的废话；每条假设需有 justification |
| 建模的创造性 | 高 | 多模型对比、创新方法、非标准解法均加分 |
| 结果的正确性与合理性 | 高 | 数值物理合理，约束不违反，限制情形下行为符合预期 |
| 表述的清晰性 | 中 | 逻辑连贯、可视化专业、读者能复现 |

## 可视化要求

- 获奖论文图表占比通常高达 **68%**；纯文字论文很难获得高奖。
- 图表必须高清矢量格式（PDF/SVG/EPS），禁止截图模糊。
- 每张图必须有说明性 caption，并在正文中明确引用和解释。
- 颜色方案应在灰度打印下仍能区分，考虑色盲友好配色。
- 每个建模问题建议 4-6 张图，敏感性分析 2-3 张，EDA 2-3 张，全文合计 13-18 张。

## 题型规律

### A 题（连续型）

- **特征**：偏微分方程、物理/工程建模、优化
- **要求**：数学推导要求高，需要扎实的微积分和数值分析基础
- **典型方法**：ODE/PDE、有限差分/有限元、优化算法、数值仿真

### B 题（离散型）

- **特征**：图论、调度、组合优化、排队论
- **要求**：算法实现要求高，需要编程能力
- **典型方法**：遗传算法、模拟退火、动态规划、网络流、启发式算法

### C 题（数据洞察型）

- **特征**：数据分析、机器学习、时序预测、可视化
- **要求**：数据处理能力要求高，需要 EDA 和模型对比
- **典型方法**：回归/分类/聚类、时间序列、SHAP 解释、多模型对比

### D/E/F 题（ICM）

- **特征**：运筹学、可持续性、政策分析
- **要求**：跨学科和文字论证要求高，需要综合多领域知识
- **典型方法**：多目标优化、系统动力学、政策建模、可持续性指标

## 高频"王炸"组合

| 问题类型 | 推荐组合 | 说明 |
| --- | --- | --- |
| 综合评价 | AHP + TOPSIS 或 AHP + 熵权 + TOPSIS | 主客观结合更有说服力 |
| 预测+不确定性 | XGBoost/随机森林 + Bootstrap 置信区间 | 预测+置信区间 |
| 风险评估 | Logistic 回归 + 蒙特卡洛模拟 | 概率+模拟 |
| 时空预测 | ARIMA/LSTM + 空间聚类 | 时间+空间维度 |
| 投资/资源优化 | 预测模型 + 动态规划/遗传算法 | 预测+优化 |
| 状态建模 | 马尔可夫链/HMM + 随机森林验证 | 状态转移+验证 |
| 文本分析（C 题）| TF-IDF + LDA 主题模型 + 情感分析 | 特征提取+主题+情感 |

## 论文结构规范

### 摘要写作要点

- 摘要是评委最先也最重视的部分，应最后撰写（内容全部确定后）。
- 每个子问题必须覆盖：用了什么方法、建立了什么模型、主要结果是什么（含关键数值）。
- 不要出现复杂公式和表格；不要只重述题目；不要用"我们认为……""本文将……"等空泛表述。
- Summary Sheet 是独立一页，应包含方法亮点、关键结论和模型创新点，不是摘要复制粘贴版。

### 标准章节结构

1. **摘要/Summary**：方法 + 结论 + 数值结果
2. **问题重述与分析**：不是照抄题面，而是建模者的理解和重新表达
3. **模型假设**：每条假设必须有必要性说明和在正文中的引用，不能是空洞罗列
4. **符号说明**：所有后文用到的符号在此定义，有量纲标注
5. **模型建立与求解**：每个子问题独立成节；先说明建模思路，再给出数学表达，再说明求解方法
6. **结果分析与检验**：数值结果 + 约束验证 + 误差分析
7. **灵敏度分析**：关键参数扰动时结果如何变化，说明模型鲁棒性
8. **模型评价与推广**：优缺点、改进方向、推广场景
9. **参考文献**：只引用真实存在的文献，格式规范

### 灵敏度分析规范

- 灵敏度分析不是可选项，是竞赛论文的必要内容。
- 方法：单参数扰动（±10%、±20%）观察目标函数变化；或 Tornado 图展示参数影响力排序。
- 对优化类问题：改变约束右端项或目标系数，观察最优解是否发生质变。
- 对预测类问题：改变关键假设或初始条件，观察预测曲线漂移幅度。
- 结论：若小扰动导致结果大幅变化，说明模型对该参数敏感，需在论文中声明。

## 常见写作错误

| 错误 | 后果 | 修正 |
| --- | --- | --- |
| 假设没有在正文建模处引用 | 假设形同虚设，评委质疑合理性 | 每条假设在首次使用时引用 |
| 结论只给数值，不解释含义 | "最优值为 1234.56"无业务意义 | 说明对应什么方案、有什么业务含义 |
| 图表出现但正文没有引导和解释 | 评委不知道图要说明什么 | 每张图前有引导句，后有 ≥3 行分析 |
| 连续三张以上图表中间没有文字 | 论文变成图集 | 每张图之间至少一段文字 |
| 公式符号首次出现处没有定义 | 读者无法理解模型 | 符号表或首次出现时定义 |
| 参考文献是捏造或无法核实的 | 严重信誉问题 | 只引用真实文献，用 Google Scholar 验证 |
| 摘要和正文数值不一致 | 摘要最后写、从正文复制数值可避免 | 写完正文后再写摘要 |

## MCM/ICM 论文验收标准

### 硬错误（直接扣分）

- 缺少 Summary Sheet
- 团队控制号未出现或出现位置错误
- 论文超过 25 页
- 使用 AI 但未附声明
- 参考文献缺失或格式混乱

### 警告（酌情扣分）

- 假设缺乏 justification
- 模型选择未与备选方案对比
- 灵敏度分析不充分
- 图表缺少 caption 或 caption 过于简略
- 论文逻辑跳跃、章节间缺乏过渡
