# 假设撰写规范

## 概述

假设是数学建模的基石。好的假设既能让问题可解，又能保持结果的合理性。竞赛评委通过假设的质量判断选手的建模思维。

## 假设分类

### 1. 必要假设（Necessary Assumptions）

**定义**：没有这些假设，问题无法建模。

**特点**：
- 通常是问题本身的隐含条件
- 必须明确写出
- 对结果有根本性影响

**示例**：
- 系统处于稳态（非随时间变化）
- 材料均匀各向同性
- 忽略次要物理效应

### 2. 简化假设（Simplifying Assumptions）

**定义**：降低问题复杂度，使模型可解。

**特点**：
- 基于工程经验或物理直觉
- 需要论证简化带来的误差可接受
- 可用于灵敏度分析

**示例**：
- 将三维问题简化为二维
- 将非线性关系近似为线性
- 将分布参数简化为集中参数

### 3. 理想化假设（Idealizing Assumptions）

**定义**：现实不可能但数学上可行的假设。

**特点**：
- 通常涉及理想化物理模型
- 需要说明偏离现实的程度
- 为数学推导提供便利

**示例**：
- 质点假设（忽略物体尺寸）
- 无摩擦假设
- 连续介质假设

## 假设格式模板

每条假设应包含三部分：**内容 + 合理性论证 + 影响说明**。

### 标准格式

```markdown
**假设N**：[假设内容]

**合理性**：[为什么这个假设是合理的]
- 依据1：[理论依据或经验依据]
- 依据2：[数据支持或文献支持]
- 依据3：[实际场景验证]

**影响**：[这个假设对模型结果的影响]
- [对精度的影响]
- [对复杂度的影响]
- [是否可用于灵敏度分析]
```

### 简化格式（适合空间有限时）

```markdown
1. 假设[内容]。合理性：[简要论证]。影响：[简要说明]
```

## 合理性论证方法

### 方法一：理论依据

基于已知的物理定律、数学定理、工程原理。

**示例**：
```markdown
**假设**：材料为线弹性体，应力与应变成正比。

**合理性**：根据胡克定律，在小变形范围内（应变 < 1%），金属材料的应力-应变关系近似线性。本问题中最大应变为 0.3%，满足线性假设条件。
```

### 方法二：经验依据

基于工程实践中的常见做法。

**示例**：
```markdown
**假设**：交通流密度在高峰期保持恒定。

**合理性**：根据交通工程学的实践经验，城市主干道在早高峰（7:00-9:00）和晚高峰（17:00-19:00）时段，交通流密度波动通常在 ±10% 以内，可近似为恒定。
```

### 方法三：数据支持

基于题目给出的数据或补充数据。

**示例**：
```markdown
**假设**：温度变化范围为 [20°C, 30°C]。

**合理性**：根据题目附表 1 中的历史数据，该地区夏季气温主要集中在 22°C-28°C 范围内，假设的温度范围覆盖了 95% 以上的实际观测值。
```

### 方法四：文献支持

引用相关文献中的类似假设。

**示例**：
```markdown
**假设**：忽略空气阻力对球体运动的影响。

**合理性**：参考文献 [3] 指出，当球体速度小于 10 m/s 时，空气阻力与重力的比值小于 1%，可忽略不计。本问题中球体最大速度约为 5 m/s，满足条件。
```

## 常见假设错误

### 1. 假设过于简化

**错误示例**：
```markdown
**假设**：问题可以简化为线性模型。

**问题**：没有论证为什么可以线性化，也没有说明线性化的误差。
```

**正确示例**：
```markdown
**假设**：在温度变化范围 [20°C, 30°C] 内，材料属性可近似为线性。

**合理性**：根据材料手册 [5]，该材料在此温度范围内的弹性模量变化为 2.3%，热膨胀系数变化为 1.8%，线性近似的误差在 3% 以内，工程上可接受。

**影响**：线性化后可使用解析求解，大幅降低计算复杂度。但当温度超出范围时，需使用非线性模型。
```

### 2. 假设未论证合理性

**错误示例**：
```markdown
**假设**：忽略摩擦力。

**问题**：没有说明为什么可以忽略，也没有量化忽略带来的误差。
```

**正确示例**：
```markdown
**假设**：忽略接触面的摩擦力。

**合理性**：本问题中接触面为光滑金属表面，摩擦系数 $\mu \approx 0.01$，而驱动力 $F = 100$ N，摩擦力 $f = \mu mg \approx 0.5$ N，仅占驱动力的 0.5%，可忽略。

**影响**：忽略摩擦力后，系统能量守恒，可使用能量法求解。误差约 0.5%。
```

### 3. 假设太多或太少

**太多**：过度简化导致模型失去实际意义。
**太少**：问题太复杂无法求解。

**建议**：5-10 条假设为宜，每条都要有明确的合理性论证。

### 4. 假设之间矛盾

**错误示例**：
```markdown
**假设1**：系统处于稳态（不随时间变化）。
**假设2**：考虑系统的时间演化过程。

**问题**：两个假设相互矛盾。
```

**正确做法**：检查假设之间的一致性，确保逻辑自洽。

### 5. 假设与结论不一致

**错误示例**：
```markdown
**假设**：忽略空气阻力。
**结论**：考虑空气阻力的修正模型更准确。

**问题**：假设中忽略了空气阻力，结论中又考虑它，逻辑矛盾。
```

**正确做法**：如果结论需要考虑被忽略的因素，应在灵敏度分析中讨论。

## 假设撰写检查清单

- [ ] 每条假设都有明确的合理性论证
- [ ] 假设之间没有逻辑矛盾
- [ ] 假设数量适中（5-10 条）
- [ ] 关键假设标注了对结果的影响
- [ ] 可用于灵敏度分析的假设有标记
- [ ] 假设的表述清晰、无歧义
- [ ] 假设基于事实或理论依据，不是随意编造
