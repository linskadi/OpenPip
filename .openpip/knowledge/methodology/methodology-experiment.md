# 实验设计方法

## 实验环境模板
```
硬件：CPU型号、GPU型号及显存、内存
软件：操作系统、Python版本、框架版本
数据集：名称、规模、划分比例
```

## 实验方案
1. **消融实验**：验证各组件贡献
2. **对比实验**：与≥3种基线方法对比
3. **参数敏感性**：分析关键参数影响
4. **鲁棒性测试**：不同噪声/工况下性能

## 评价指标
| 指标 | 公式 | 适用 |
|------|------|------|
| 准确率(ACC) | (TP+TN)/(TP+TN+FP+FN) | 分类 |
| 精确率(Precision) | TP/(TP+FP) | 不平衡数据 |
| 召回率(Recall) | TP/(TP+FN) | 故障检测 |
| F1 | 2·P·R/(P+R) | 综合 |
| MSE | Σ(y-ŷ)²/n | 回归 |
| RMSE | √MSE | 回归 |

## 结果展示
- 折线图：趋势变化
- 柱状图：对比结果
- 热力图：相关性/混淆矩阵
- 箱线图：分布和异常值
- 表格：精确数值对比

## 结果描述模板
"实验结果表明，所提方法在[指标]上达到[数值]，较最优基线[方法名]提升[百分比]。"

---

## 实验设计原则（建设性指导）

### 基线选择三原则
1. **Naive baseline**：最简单的合理方法——建立性能下界
2. **SOTA baseline**：当前最强方法（至少 2 个）——建立比较基准
3. **Ablation baseline**：自身方法的简化版——证明每个组件的贡献

### 实验分析的四个层次
1. **描述层**：结果是什么（"方法 A 在数据集 X 上达到 95.2%"）
2. **比较层**：与基线对比（"比 SOTA 高 2.3%"）
3. **解释层**：为什么（"因为组件 B 解决了 C 问题"）
4. **推论层**：意味着什么（"表明 D 在 E 条件下是关键因素"）

大部分论文只做到 1-2 层。好论文做到 3-4 层。

### 消融实验设计
- 每个方法组件至少 1 个消融
- 消融顺序：先移除最重要的组件
- 报告消融后的性能变化 + 分析为什么

### 错误分析
- 展示失败案例（不只是成功案例）
- 分析错误模式（哪些类型的数据/场景容易出错）
- 讨论如何改进
