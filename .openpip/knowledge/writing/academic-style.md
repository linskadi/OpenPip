# 学术写作规范

## 语言
- 第三人称（本文、本研究）
- 被动语态为主
- 禁止"我"、"我们"、"你"

## 段落
每段：主题句→论据→过渡句

## 引用
- 每段2-4篇
- 叙事引用："张三等[1]提出..."
- 括号引用："已有研究[1,2]表明..."

## 时态
- 背景：现在时
- 方法：现在时或过去时
- 实验：过去时
- 结论：现在时

## 好/坏对比
| ❌ 不好 | ✅ 好 |
|---------|-------|
| 我们做了一个实验 | 本研究开展了实验验证 |
| 这个方法很好 | 该方法具有较高的准确率 |
| 很多学者研究过 | 已有大量学者对此展开研究[1-3] |
| 结果证明了我们的想法 | 实验结果验证了上述假设 |
| 所以这个方法是最优的 | 实验结果表明该方法在综合指标上优于对比方法 |

## 常见错误
1. 连续用"首先...其次..." → 删除排列式
2. 每段开头"本文..." → 变化句式
3. 公式后无变量定义 → 补充"其中，$x$ 为..."
4. 图表无正文引用 → 补充"如图1所示"
5. 引用密度不均 → 每段2-4篇

## 建设性指导（如何写好）

### 如何写有说服力的段落
1. **主题句要具体**：不要"本章介绍了方法"，而是"本章提出基于X的Y方法，解决Z问题"
2. **论据要量化**：不要"性能有所提升"，而是"准确率从87.3%提升至92.1%"
3. **论证链要完整**：Evidence → Warrant → Claim，中间不能跳步
4. **过渡要自然**：用逻辑连接词（因此、然而、此外），不要硬切

### 如何管理读者预期
- 每章开头告诉读者"本章要做什么"
- 每章结尾告诉读者"本章发现了什么"
- 复杂概念先给直觉再给定义
- 公式前用一句话说明"为什么需要这个公式"

### 如何处理复杂概念引入
1. 先用类比或日常经验建立直觉
2. 给出严格定义
3. 用简单例子演示
4. 再推广到一般情况
